Depot — Fork Plan
The skill-first, enterprise-ready agent interface.
A fork of Craft Agents (Apache 2.0) focused on visual skill dispatching and AWS Bedrock support.
1. Why Fork (Not Patch)
Craft Agents is session-centric — you open a session, then figure out what to do. Depot is skill-centric — you see your capabilities, pick one, and a session is created with the right context pre-loaded. This is a UX-level divergence, not a feature addition.
Craft AgentsDepotPrimary nav: session inboxPrimary nav: skill catalog"Start a session, then @mention a skill""Click a skill, session is created for you"Anthropic API / OpenRouter / CopilotAdds AWS Bedrock as first-class providerGeneral-purpose agent interfaceOptimized for enterprise devs with defined workflowsSkills are an add-onSkills are the core abstractionUpstream relationship: Depot should track Craft Agents upstream for core agent SDK updates, session management, and MCP infrastructure. The fork diverges on navigation UX, provider support, and skill-first workflows.
2. Target User
Enterprise developers who:

Run Claude through AWS Bedrock (org policy)
Have 3-10 well-defined agent skills (PM, DevOps, QA, KB, etc.)
Want visual, one-click dispatching instead of remembering slash commands
Need session history organized by skill, not by time
Not for: people who want a general chat interface, or multi-agent orchestration/swarm systems.

Each skill card shows:

Skill name + icon
Quick Commands — buttons that launch a session with a pre-filled prompt template
Last used timestamp
Session count badge
Clicking a Quick Command:

Creates a new session
Pre-loads the skill's system context (project config, MCP sources, etc.)
Optionally pre-fills the first message from a prompt template
Opens the session view (standard Craft Agents session UI)
3.2 Skill Definition Format
Extend Craft Agents' existing skill format with a depot.yaml manifest:

# ~/.depot/skills/pm/depot.yaml
name: Product Manager
icon: clipboard        # lucide icon name
provider: bedrock      # which LLM connection to use
description: "Sprint planning, ticket refinement, backlog health for Project Phoenix"

sources:               # MCP servers / APIs to auto-connect
  - jira-mcp
  - confluence-mcp

quick_commands:
  - name: Refine Ticket
    template: "Refine ticket {{ticket_id}} — check acceptance criteria, sizing, and dependencies"
    variables:
      ticket_id:
        prompt: "Ticket ID (e.g. PROJ-1234)"
        type: string

  - name: Sprint Planning
    template: "Plan the next sprint. Review current velocity, unfinished items, and top-priority backlog items."

  - name: Backlog Health
    template: "Analyze backlog health — find stale tickets, duplicates, and items missing sizing or priority."

context_files:         # files loaded into skill context
  - ./project-context.md
  - ./backlog-strategy.md
This is additive — standard Craft Agents SKILL.md files still work, depot.yaml just adds the visual/dispatch layer.

3.3 AWS Bedrock Provider
Add a new LLM connection type alongside Anthropic/OpenRouter/Copilot:

Connection Type: AWS Bedrock
─────────────────────────
Region:          [us-east-1        ▼]
Auth:            [SSO / IAM Role   ▼]
Model:           [claude-sonnet-4-20250514  ]
Profile (opt):   [my-org-profile   ]
Implementation notes:

Use @aws-sdk/client-bedrock-runtime with InvokeModelWithResponseStream
Support IAM role assumption and SSO credential chains
Map Bedrock's response format to the internal message format Craft Agents already uses
Per-workspace default: enterprise workspace uses Bedrock, personal workspace uses Anthropic API

Skill badge on each session in the list
Search across sessions within a skill
4. What NOT to Build (Keep from Upstream)
Session UI — Craft Agents' session view, streaming, tool visualization: keep as-is
MCP infrastructure — source connections, stdio/SSE transports: keep as-is
Permission modes — Explore / Ask to Edit / Execute: keep as-is
File handling — drag-drop, document conversion: keep as-is
Workspace system — isolation, themes, config: keep as-is
Multi-agent / swarm — explicitly out of scope for v1

Positioning
"Depot is the skill-first agent interface for enterprise teams. If you run Claude through AWS Bedrock and organize your work around defined agent capabilities, Depot gives you a visual command center to dispatch and track your skills."

Differentiator from Craft Agents
Bedrock support (enterprise unlock)
Skill-first UX (opinionated, not general-purpose)
Quick Commands (declarative dispatch, not just chat)
Skill-grouped history
